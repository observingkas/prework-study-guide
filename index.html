CSS

  • A selector (e.g., div) targets the elements or attributes to which the rules apply.
  • Declarations have two parts: the CSS property and its value. For example, color: blue; has color as the property and blue as its value.
  • In CSS, the margin property adds space outside an element (not visible on the page), while the padding property adds space inside the element, making it larger. The CSS box model includes margins, borders, padding, and content.
  • The * selector applies styles to all elements on the page.
  • We combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors.
  • To connect an external CSS file to an HTML file, we use the element. Having both files in the same project doesn’t automatically link them.
  • We add a link element to create a connection to an external file, and add it after the final meta element but before the title element.
  • The link element connects to an external file. Add it after the last meta element and before the title element
  • The rel attribute defines the relationship between the current document and the linked resource.
  • h1 and h2 rule assignment is a positioning property, text-align. This is one way to center a title or heading on a webpage.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • box-shadow: Adds shadow effects.
  • X offset (#px): Defines the shadow's horizontal extent (to the right).
  • Y offset (#px): Defines the shadow's vertical extent (below the box).
  • Color: Defined using an HTML hexadecimal code (e.g., #888888).
  • Multiple shadows can be set by separating them with commas.
  • width: #%;: Sets the width to a #% of the container.
  • margin: #px auto;: Adds a #px margin on top and bottom, and centers horizontally.
  • border: #px solid gray;: Adds a #px solid gray border.
  • box-shadow: #px #px ##;: Adds a shadow with a #px horizontal offset, #px vertical offset, and color ##.
  • The href attribute specifies the URL of the external resource. It uses a relative path based on the current document’s location (e.g., index.html). To find the stylesheet, we look in the assets folder, so the path is assets/style.css.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout -b feature/starter-code: Creates a feature branch named "starter code" -or whatever you'd want to replace that with-
  • The git command must precede every git statment
  • The checkout command is to move the working branch to a new branch
  • The -b flag creates a new branch
  • The feature/starter-code is the name of the new branch. The prefix feature indicates it is going to be a feature branch
  • Git add -A: Use this to add the starter code to the current working branch. The add command adds modifications in the current working branch to the staging area. The -A flag indicates that we want to add all changes.
  • The staging area informs Git that the changes will be added in the next commit
  • git commit -m "add starter code": A commit that creates a snapshot of the currently staged changes. It adds to the current branch you are in.
  • Use git pull origin main to double check if our local branch is in sync with the base branch in GitHub. It is important to always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work. Otherwise, you might be working on outdated code

JavaScript

  • Variable Types: String: Sequence of text; use quotes. | Number: Numeric values; no quotes. | Boolean: True/false values; use keywords without quotes. | Array: Stores multiple values in one reference. | Object: Any data structure; everything in JavaScript is an object. |
  • Conditionals: Allow execution of code based on true/false evaluations. The if...else statement checks a condition: if true, the if block runs; if false, the else block executes. This enables dynamic decision-making in your code. You can also use else if to chain multiple conditions, creating complex logic flows to handle various scenarios effectively.
  • Functions: Packages reusable code; defined with function. Example: function helloWorld() then add curly brackets and code. Call functions by name to execute.
  • Events: Code structures that respond to browser activities (e.g., clicks).
  • Declaring Variables: Use const for immutable variables; let for variables that can change.
  • Linking Files: link for CSS in head, script for JavaScript in .
  • Control Flow: Code execution order, top to bottom. Conditional statements and loops control flow.
  • Truthy and Falsy Values: Falsy: 0, -0, "", null, undefined, NaN, false. Truthy: All other values.
  • Arrays: Group related data. Declared using const, e.g., const shapes = ["triangle", "square", "pentagon", "circle"];.
  • For Loops: Efficiently iterate through array items.
  • Debugging Functions: Ensure functions are called to execute. Update function references as needed.
